# Code of Conduct

## The short version

Be a decent person. Treat everyone the way you'd want to be treated if you were new to a project and trying to contribute.

---

## What this project is

This is a developer tool built in the open. People come here to report bugs, ask questions, suggest ideas, and contribute code. Most of them are learning something, about NestJS, about software architecture, about open source. That's worth respecting.

---

## What's expected

**Be patient.** Not everyone has the same background. Someone asking a question that seems obvious to you might be a junior developer making their first open source contribution. That's a good thing. Help them, or point them somewhere useful.

**Be honest.** If a PR isn't good enough to merge, say so clearly and explain why. Vague rejections like "this doesn't fit the project" without explanation aren't fair to the person who put in the work. Honest feedback, delivered kindly, is more respectful than a soft rejection.

**Be specific.** "This is wrong" isn't useful. "This breaks the dependency rule in the domain layer, infrastructure imports shouldn't appear here" is. Back up your feedback with reasoning.

**Assume good intent first.** If a comment reads as hostile, it's probably just blunt. Most developers write like they're in a hurry, that's not the same as being rude. Read charitably before reacting.

**Give credit.** If someone's idea inspired your solution, say so. If you're building on someone else's PR, reference it.

---

## What isn't welcome

There's a short list of things that will get you removed from this project immediately, with no warning:

- Harassment of any kind, targeted, persistent, or personal
- Discrimination based on gender, race, nationality, religion, sexuality, disability, age, or anything else that has nothing to do with the code
- Doxxing or sharing someone's personal information without their consent
- Threats, real or implied

Everything else is handled with context. A bad day, a blunt comment, an honest mistake, those get a conversation, not a ban. Patterns of behavior that make people feel unwelcome get addressed more seriously.

---

## When something goes wrong

If someone's behavior in this project is making your experience genuinely bad, reach out privately before escalating. Most things get resolved faster that way.

If you need to report something to the maintainer directly, especially if it involves harassment or safety, email is the right channel. The address is in the repository's `SECURITY.md`. Reports are handled confidentially. You won't be asked to justify yourself publicly.

---

## Scope

This applies everywhere this project has a presence: GitHub issues, pull requests, discussions, and any community spaces that get created in the future.
