# Security Policy

## Supported Versions

Currently, only the latest generic release is officially supported with security updates.

| Version | Supported |
| ------- | --------- |
| v0.x    | Yes       |
| < v0.1  | No        |

## Reporting a Vulnerability

I take the security of Nest-Pilot very seriously. If you discover a security vulnerability, I appreciate your help in disclosing it to us in a responsible manner.

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

Instead, please report them using one of the following methods:

1. **GitHub Private Vulnerability Reporting**:
   Navigate to the [Security advisory tab](https://github.com/Mehedi-Hasan0/nest-pilot/security/advisories) in this repository and click on **"Report a vulnerability"**. This is the preferred method as it allows us to track and collaborate on a fix securely.

2. **Email**:
   If you cannot use GitHub's private reporting tool, please email the maintainers directly at `developermahedi.me@gmail.com` with the subject line "SECURITY VULNERABILITY: Nest-Pilot".

### What to Include in Your Report

Please provide a detailed report including:

- A description of the vulnerability and its potential impact.
- Exact steps to reproduce the issue.
- (Optional but appreciated) A proof-of-concept (PoC) or suggested mitigation.

### Response Timeline

I will acknowledge receipt of your vulnerability report within 48 hours and strive to provide a timeline for a resolution. I will coordinate with you to publish a security advisory once the patch is released.
